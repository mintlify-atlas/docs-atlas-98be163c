`PS4Server` is the local HTTP server at the heart of DirectPackageInstaller. It starts on port 9898 (the `Installer.ServerPort` constant) and accepts connections from both the PS4 package downloader and the GoldHEN payload client. Every route handler is an `async Task` method; all routes inspect the incoming `Range` header and respond with `206 Partial Content` when a range is present ‚Äî a requirement of the PS4's built-in package installer.

## Server startup

`PS4Server` is constructed with a local IP and port and backed by `HttpServerLite.Webserver`. All inbound requests are handled by a single default route dispatcher (`Process`):

```csharp
public PS4Server(string IP, int Port = 9898)
{
    Server = new Webserver(new WebserverSettings(IP, Port) { ‚Ä¶ });
    Server.Routes.Default = Process;
}
```

The `Process` method strips the query string, identifies the leading path segment, and dispatches to the appropriate handler. It also detects whether the requester is the PS4's native downloader (indicated by a double `?` in the URL) versus Remote Package Installer.

## URL parameter encoding

All routes that accept a target URL support two query string formats:

- `?url=<percent-encoded-url>` ‚Äî plain URL encoding
- `?b64=<base64-encoded-url>` ‚Äî Base64 encoding of the UTF-8 URL bytes

The `b64` form is used when the URL itself contains characters that would be mangled by a second round of percent-encoding (common when the URL is itself a DPI server URL).

## Route reference

### `/proxy/?b64=<base64url>`

Fetches the decoded URL through DPI and streams the response to the PS4. Handles `Range` requests for partial content.

```
PS4 ‚îÄ‚îÄRange GET‚îÄ‚îÄ‚ñ∂ /proxy/?b64=‚Ä¶ ‚îÄ‚îÄ‚ñ∂ FileHostStream(url) ‚îÄ‚îÄ‚ñ∂ PS4
```

If the resolved host has `SingleConnection = true` (i.e., `FileHostBase.Limited`), the stream is wrapped in a `ReadSeekableStream` to serialise seek operations before passing it to `SendStream`.

### `/cache/?b64=<base64url>`

Serves data from a `DownloaderTask` segmented download cache. The route waits until the cache reports a non-zero length before sending any data.

**Back-pressure protection:** if the PS4 seeks to a position more than 100 MB ahead of the current safe-read boundary (`MaxSkipBufferSize = 100 MB`) and this is not the first connection for this URL, the route returns:

```http
HTTP/1.1 429 Too Many Requests
Connection: close
Retry-After: 300
```

This prevents the PS4 from outrunning the background download.

### `/file/?b64=<base64url>`

Serves a local file by decoded path. Opens the file with `FileShare.Read` so other processes (or a second DPI connection) can read it concurrently.

### `/merge/?b64=<base64url>`

Merges the pieces of a PSN split-PKG JSON manifest into a single virtual stream. The decoded URL may point to either a local JSON file or a remote URL. The merged stream is then served to the PS4 as a single contiguous byte sequence.

### `/split/?b64=<base64url>&offset=N&size=N`

Serves a fixed-length byte-range slice of a source. Used by the `PKGManifest` pieces generated by `RegisterJSON` when `AutoSplit` is enabled:

```
GET /split/?b64=<url>&offset=4294967296&size=4294967296
```

The handler opens the source (local file or remote URL), wraps it in a `VirtualStream(source, offset, size)`, and passes it to `SendStream`. If the source host has `SingleConnection = true`, a `ReadSeekableStream` wrapper is applied before slicing.

### `/json/{id}.json`

Serves a `PKGManifest` JSON document registered in memory by `RegisterJSON`. The `{id}` path segment indexes into the `PS4Server.JSONs` dictionary. The document is served with `Content-Type: application/json`.

**Example response:**

```json

  "packageDigest": "‚Ä¶",
  "numberOfSplitFiles": 2,
  "pieces": [
    {
      "url": "http://192.168.1.100:9898/split/?b64=‚Ä¶&offset=0&size=4294967296",
      "fileOffset": 0,

      "hashValue": "0000000000000000000000000000000000000000"
    },
    {
      "url": "http://192.168.1.100:9898/split/?b64=‚Ä¶&offset=4294967296&size=4294967296",
      "fileOffset": 4294967296,

      "hashValue": "0000000000000000000000000000000000000000"

```

### `/unrar/?id=<id>` and `/un7z/?id=<id>`

Stream a decompressed archive entry identified by `id`. The `id` maps to a `DecompressService.TaskCache` entry that holds the source URL and entry filename. Decompression runs in a background task; `PS4Server` reads from the decompressed stream and pipes it to the PS4.

<Note>
  RAR and 7z decompression always happen in streaming (piped) mode ‚Äî the full
  decompressed content is never written to disk unless the host is `Limited`
  (disk cache mode). Download, decompression, and upload to the PS4 happen
  concurrently.
</Note>

## GoldHEN payload protocol

When neither RPI nor etaHEN is detected on the console, DPI uses the GoldHEN payload path implemented in `PayloadService` (`Host/PayloadService.cs`).

### Step 1 ‚Äî Connect to the payload port

`TryConnectSocket` attempts a TCP connection to the PS4, trying ports in order:

```
9090 ‚Üí 9021 ‚Üí 9020
```

Each attempt has a 3-second timeout. If all three ports fail, the method retries once after a 3-second delay before giving up.

### Step 2 ‚Äî Patch and send the payload binary

`EnsureClient` patches the compiled payload binary (`Resources.Payload`) at the marker offset before sending it:

```csharp
var Offset = Payload.IndexOf(new byte[] { 0xB4, 0xB4, 0xB4, 0xB4, 0xB4, 0xB4 });
// Writes 4 bytes of PC IP at Offset
// Writes 2 bytes of big-endian port at Offset + 4
IP.CopyTo(Payload, Offset);
Port.CopyTo(Payload, Offset + 4);     // Port is reversed (big-endian)
PayloadSocket.Send(Payload);
```

The six `0xB4` sentinel bytes mark the location where the PC's IP address (4 bytes) and the PKG info server port (2 bytes, big-endian) are embedded at runtime. The PS4 executes this payload and connects back to the PC on the patched address and port.

### Step 3 ‚Äî Accept the callback connection

`ServiceSocket` listens on an ephemeral local port (or a fixed port from `App.Config.PayloadPort`). The server loop (`ServerLoop`) calls `AcceptAsync` with a 10-second cancellation token and queues each accepted socket. `SendPKGPayload` dequeues the first socket within a 10-second window; if none arrives, it reports failure.

### Step 4 ‚Äî Send the PKG info packet

Once the PS4 has connected back, DPI sends a binary PKG info packet over that socket:

After sending, both sockets (`PKGInfoSocket` and `PayloadSocket`) are closed. The PS4 uses the embedded URL to start downloading the PKG through DPI's HTTP server.

<Tip>
  To send a "service exit" signal instead of a new package, set the message type
  field to `0` instead of `1`. `StopServer` uses a 4-byte zero buffer for this
  purpose when draining the connection queue on shutdown.
</Tip>

## `SendStream` internals

All route handlers delegate to the private `SendStream` method, which:

1. Sets `Content-Length` from `Origin.Length`.
2. Sets `Connection: Keep-Alive` and `Accept-Ranges: none`.
3. If a `Range` was requested, wraps `Origin` in a `VirtualStream(origin, begin, length)` and writes the `Content-Range` header.
4. Wraps the (possibly sliced) stream in a `BufferedStream`.
5. Calls `Context.Response.SendAsync(length, stream, token)`.

<Warning>
  `Accept-Ranges: none` is set deliberately. DPI manages range slicing itself
  via `VirtualStream` ‚Äî it does not advertise byte-range support to prevent
  the PS4 from issuing concurrent range requests that would conflict with
  streaming sources.
</Warning>
